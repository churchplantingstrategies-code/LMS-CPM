## 🔧 What YOU Need To Do

### 1. Get Google OAuth Credentials (5 minutes)

Follow [GOOGLE_OAUTH_SETUP.md](./GOOGLE_OAUTH_SETUP.md) to:
1. Create a Google Cloud project
2. Enable Google+ API
3. Create OAuth 2.0 credentials
4. Copy your `CLIENT_ID` and `CLIENT_SECRET`

### 2. Add Credentials to `.env.local` (1 minute)

Open `.env.local` and fill in:

```bash

GOOGLE_CLIENT_SECRET="paste-your-client-secret-here"
```

### 3. Restart Dev Server (30 seconds)

```powershell
# Kill current server (Ctrl+C)
npm run dev
```

### 4. Test It (1 minute)

Go to http://localhost:3000/login
- Click **"Continue with Google"**
- Sign in with your Google account
- Should redirect to dashboard with auto-created account

## 🔐 Security Notes

**For Local Development:**
- Use Google test credentials (optional)
- `http://localhost:3000` is fine
- No HTTPS needed

**For Production:**
1. Create separate Google OAuth credentials
2. Use your production domain (e.g., `https://yourdomain.com`)
3. Add to Google Console:
   - Authorized origins: `https://yourdomain.com`
   - Redirect URIs: `https://yourdomain.com/api/auth/callback/google`
4. Store credentials in production `.env`
5. Ensure NEXTAUTH_SECRET is very strong: `openssl rand -base64 32`

## 🐛 Common Issues & Fixes

| Issue | Solution |
|-------|----------|
| "Invalid client" error | Check GOOGLE_CLIENT_ID/SECRET are copied exactly (no spaces) |
| "Redirect URI mismatch" | Ensure `http://localhost:3000/api/auth/callback/google` is in Google Console |
| Google button doesn't appear | Restart dev server after setting env vars |
| New user not created | Check Prisma is connected and database is initialized |
| "Callback URL mismatch" | Make sure NEXTAUTH_URL="http://localhost:3000" in .env.local |
